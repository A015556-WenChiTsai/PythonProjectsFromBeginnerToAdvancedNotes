# Python Terminal Maze Solver (BFS Visualization)

這是一個基於 Python `curses` 模組的迷宮路徑搜尋視覺化程式。它使用 **廣度優先搜尋演算法 (Breadth-First Search, BFS)** 來尋找從起點 'O' 到終點 'X' 的最短路徑，並在終端機中動態展示搜尋過程。

## 🚀 專案特點 (Key Features)

- **演算法視覺化**：即時顯示演算法如何探索迷宮。
- **最短路徑保證**：使用 BFS 演算法，確保找到的路徑是最短的。
- **終端機介面 (TUI)**：使用 `curses` 繪製圖形，無需安裝龐大的 GUI 函式庫。
- **資料結構應用**：實作了 Queue (佇列) 與 Set (集合) 來優化搜尋效率。

## 🛠️ 技術棧 (Tech Stack)

- **語言**：Python 3.x
- **核心模組**：
  - `curses`: 用於終端機圖形繪製。
  - `queue`: 實作 FIFO 佇列，用於 BFS。
  - `time`: 控制動畫速度。

## 📋 前置需求 (Prerequisites)

- Python 3.6 或更高版本。
- **注意**：`curses` 模組通常內建於 Linux 和 macOS 的 Python 中。
  - **Windows 使用者**：標準 Python 安裝可能不包含 `curses`。你需要安裝 `windows-curses`：


### 1. `stdscr` (Standard Screen) —— 你的畫布

*   **全名**：Standard Screen（標準螢幕視窗）。
*   **是什麼**：它是一個**物件 (Object)**，代表了你目前終端機（Terminal）的整個視窗畫面。
*   **做什麼用**：
    *   它是你用來「畫圖」或「寫字」的地方。
    *   在你的程式碼中，所有的繪圖指令都是對著它發出的。
*   **程式碼中的例子**：
    *   `stdscr.addstr(...)`：在畫布上寫字（畫迷宮的牆壁或路徑）。
    *   `stdscr.clear()`：把畫布擦乾淨。
    *   `stdscr.refresh()`：把畫好的內容真正顯示到螢幕上。

**比喻**：
想像 `stdscr` 是一塊**黑板**。你要寫字、擦黑板，都需要拿著這塊黑板操作。

---

### 2. `wrapper` —— 你的安全管理員

*   **來源**：`from curses import wrapper`
*   **為什麼需要它**：
    *   `curses` 程式會改變終端機的設定（例如隱藏游標、改變顏色、攔截鍵盤輸入）。
    *   如果程式**當機**或**出錯**（Crash），沒有正常結束，終端機就會「壞掉」（字體顏色亂掉、打字看不到字、換行錯亂），你通常得關掉重開終端機才能解決。
*   **做什麼用**：
    `wrapper` 是一個**包裝器 (Wrapper)**，它幫你處理「初始化」與「善後」的工作。
    1.  **初始化**：它會自動建立 `stdscr`，設定好顏色模式，隱藏游標等。
    2.  **執行**：它會呼叫你的 `main` 函式，並把 `stdscr` 傳進去給你看。
    3.  **善後 (最重要)**：無論你的程式是正常結束，還是因為寫錯程式碼而報錯 (Error)，`wrapper` 都保證會把終端機**恢復原狀**（還原游標、顏色等），不會讓你的終端機壞掉。

**比喻**：
想像 `wrapper` 是一個**場地管理員**。
1.  你 (main 函式) 要借教室 (終端機) 來辦活動。
2.  管理員 (`wrapper`) 先幫你開門、開燈、排好桌椅 (初始化)。
3.  然後讓你進去辦活動 (執行 `main`)。
4.  活動結束後，或者如果你在裡面不小心把東西打破了，管理員都會負責進去把垃圾清乾淨、關燈鎖門，讓教室恢復原狀 (善後)。

---

### 兩者的關係

在你的程式碼最後一行：
```python
wrapper(main)
```
這句話的意思是：
「呼叫管理員 (`wrapper`)，請他幫忙準備環境，然後執行我的主程式 (`main`)。」

當 `wrapper` 執行 `main` 時，它會自動把準備好的黑板 (`stdscr`) 塞給 `main` 函式，所以你的 `main` 函式定義才會寫成 `def main(stdscr):`。
